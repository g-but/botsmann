## Template Usage Instructions

**This template is designed to ensure consistency across all Botsmann bot documentation.**

### Required Sections (Must Include)
1. ✅ Overview (2-3 paragraphs)
2. ✅ Core Value Proposition
3. ✅ Key Features (3-5 categories with sub-features)
4. ✅ Current Capabilities
5. ✅ File Structure
6. ✅ Technical Architecture
7. ✅ Use Cases (3-4 user types)
8. ✅ Development (running, testing, building)
9. ✅ Future Roadmap (4 quarters)
10. ✅ Privacy & Security
11. ✅ Integration Possibilities
12. ✅ Contributing
13. ✅ Resources
14. ✅ Support
15. ✅ License

### Optional Sections (Add If Relevant)
- Domain-specific compliance (medical, legal, etc.)
- Educational methodology (for learning bots)
- Cultural context (for language/regional bots)
- API documentation (if bot has public API)

### Quality Targets
- **Length**: 150+ lines (excluding template instructions)
- **Detail Level**: Similar to Lex, Artr, Nerd, Imhotep examples
- **Technical Depth**: Include architecture, data flow, integrations
- **User Focus**: Clear use cases and benefits for target users
- **Future Vision**: Concrete roadmap with quarterly milestones

### Tips for Great Bot Documentation
1. **Be Specific**: Avoid generic "AI-powered" language. Explain HOW your bot is specialized.
2. **Show, Don't Tell**: Include examples, code snippets, screenshots.
3. **Think User-First**: Answer "Why should I use this?" before "How does it work?"
4. **Technical Credibility**: Show the tech stack, architecture decisions, integrations.
5. **Roadmap Clarity**: Make plans concrete with quarters and checkboxes.
6. **Domain Expertise**: Demonstrate understanding of the domain's unique requirements.

- [ ] All required sections completed
- [ ] At least 150 lines of content

- [ ] Use cases for 3+ user types
- [ ] Roadmap with 4 quarters

- [ ] Reviewed for typos and clarity
